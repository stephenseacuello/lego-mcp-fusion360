# Output Directory

This directory contains files generated by the LEGO MCP Server.

## Directory Structure

```
output/
├── stl/                    # STL mesh files
│   └── Brick_2x4.stl       # Example: 2x4 brick mesh
│
├── gcode/
│   ├── milling/            # CNC milling G-code
│   │   └── Brick_2x4.nc    # Example: GRBL-compatible G-code
│   │
│   └── 3dprint/            # 3D printing G-code
│       └── Brick_2x4_prusa.gcode  # Example: Prusa-compatible G-code
│
└── fusion_files/           # Optional: Saved Fusion 360 files
    └── Brick_2x4.f3d       # Example: Fusion 360 design file
```

## File Naming Convention

Files are named according to this pattern:

- **STL**: `{BrickType}_{StudsX}x{StudsY}.stl`
- **Mill G-code**: `{BrickType}_{StudsX}x{StudsY}_{machine}.nc`
- **Print G-code**: `{BrickType}_{StudsX}x{StudsY}_{printer}_{quality}.gcode`

## Sample Files

The `SAMPLE_*` files are example outputs for reference. They show the expected format but are not functional G-code.

## Notes

- STL files are high-resolution by default
- Milling G-code uses the post processor specified (default: GRBL)
- 3D print G-code includes print statistics in the header

## Cleaning Up

Output files can grow large. To clean up:

```bash
# Remove all generated files (keeps samples)
find output -name "*.stl" ! -name "SAMPLE*" -delete
find output -name "*.nc" ! -name "SAMPLE*" -delete
find output -name "*.gcode" ! -name "SAMPLE*" -delete
```
